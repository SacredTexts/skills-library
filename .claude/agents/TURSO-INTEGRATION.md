# Turso (libSQL) Integration for Agent Meta-Tracking

**Status**: ✅ Fully Operational
**Database**: `agents-ista` at `libsql://agents-ista.aws-ap-northeast-1.turso.io`
**Purpose**: Agent documentation, code examples, performance metrics for speed.md analysis
**Note**: This is **NOT** replacing Neon DB (main application database)

---

## Purpose

Turso is a **cloud-native edge database** (libSQL - SQLite-compatible) used exclusively for **agent meta-tracking**:

1. **Agent Documentation Changes** - Track what agents modify in their documentation
2. **Code Examples** - Store code patterns and examples generated by agents
3. **Performance Metrics** - Track execution times, bottlenecks, and optimizations
4. **Speed Agent Analysis** - Provide data for speed.md to study and improve the system

---

## Database Schema

### Tables (5 Total)

#### 1. **agent_executions**
Tracks individual agent task executions with timing and status.

```sql
CREATE TABLE agent_executions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  agent_type TEXT NOT NULL,        -- coder, tester, stuck, monitor, speed
  task_id TEXT NOT NULL,           -- Unique task identifier
  start_time INTEGER NOT NULL,     -- Unix epoch ms
  end_time INTEGER,                -- Unix epoch ms
  duration_ms INTEGER,             -- Execution duration
  status TEXT NOT NULL,            -- success, error, timeout
  error_message TEXT,              -- Error details if failed
  created_at INTEGER NOT NULL
);
```

**Indexes**: agent_type, status, created_at, task_id

#### 2. **bottlenecks**
Tracks performance bottlenecks by agent and operation type.

```sql
CREATE TABLE bottlenecks (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  agent_type TEXT NOT NULL,
  operation TEXT NOT NULL,         -- schema_create, migration_apply, etc.
  avg_duration_ms INTEGER NOT NULL,
  occurrence_count INTEGER NOT NULL,
  severity TEXT NOT NULL,          -- low, medium, high, critical
  created_at INTEGER NOT NULL
);
```

**Indexes**: agent_type, severity, created_at

#### 3. **optimizations**
Tracks optimization attempts and their effectiveness.

```sql
CREATE TABLE optimizations (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  optimization_type TEXT NOT NULL, -- caching, parallelization, etc.
  agent_type TEXT,                 -- NULL if system-wide
  before_ms INTEGER NOT NULL,      -- Duration before
  after_ms INTEGER NOT NULL,       -- Duration after
  success_rate REAL NOT NULL,      -- 0.0 to 100.0
  created_at INTEGER NOT NULL
);
```

**Indexes**: agent_type, optimization_type, created_at

#### 4. **quality_metrics**
Tracks quality metrics per agent type.

```sql
CREATE TABLE quality_metrics (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  agent_type TEXT NOT NULL,
  test_pass_rate REAL NOT NULL,      -- 0.0 to 100.0
  false_positive_rate REAL NOT NULL, -- 0.0 to 100.0
  bug_count INTEGER NOT NULL,
  created_at INTEGER NOT NULL
);
```

**Indexes**: agent_type, created_at

#### 5. **task_patterns**
Tracks recurring task patterns and their performance.

```sql
CREATE TABLE task_patterns (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  pattern_name TEXT NOT NULL,         -- e.g., "schema_create_to_migration"
  sequence TEXT NOT NULL,             -- JSON array of agent sequence
  frequency INTEGER NOT NULL,         -- Occurrence count
  avg_total_time_ms INTEGER NOT NULL, -- Average total time
  created_at INTEGER NOT NULL
);
```

**Indexes**: pattern_name, frequency, created_at

## Usage Examples

### Logging Agent Execution

```typescript
import { turso } from '@/lib/turso';

async function logExecution(agentType: string, taskId: string, durationMs: number) {
  const result = await turso.execute({
    sql: `INSERT INTO agent_executions
          (agent_type, task_id, start_time, end_time, duration_ms, status, created_at)
          VALUES (?, ?, ?, ?, ?, ?, ?)`,
    args: [
      agentType,
      taskId,
      Date.now() - durationMs,
      Date.now(),
      durationMs,
      'success',
      Date.now()
    ]
  });

  return result.lastInsertRowid;
}
```

### Logging Bottleneck

```typescript
async function logBottleneck(
  agentType: string,
  operation: string,
  avgDurationMs: number,
  severity: 'low' | 'medium' | 'high' | 'critical'
) {
  await turso.execute({
    sql: `INSERT INTO bottlenecks
          (agent_type, operation, avg_duration_ms, occurrence_count, severity, created_at)
          VALUES (?, ?, ?, ?, ?, ?)`,
    args: [agentType, operation, avgDurationMs, 1, severity, Date.now()]
  });
}
```

### Querying Performance Data

```typescript
// Get recent agent executions
const executions = await turso.execute(`
  SELECT * FROM agent_executions
  WHERE agent_type = 'coder'
  ORDER BY created_at DESC
  LIMIT 10
`);

// Analyze bottlenecks
const bottlenecks = await turso.execute(`
  SELECT agent_type, operation, AVG(avg_duration_ms) as avg_time
  FROM bottlenecks
  WHERE severity IN ('high', 'critical')
  GROUP BY agent_type, operation
  ORDER BY avg_time DESC
`);
```

## Integration with Speed Agent

The **speed agent** (`speed.md`) uses this database to:

1. **Analyze Performance** - Query agent execution times and identify slow operations
2. **Detect Bottlenecks** - Find recurring performance issues across agents
3. **Track Optimizations** - Measure effectiveness of applied optimizations
4. **Learn Patterns** - Identify common task sequences and their performance
5. **Quality Monitoring** - Ensure optimizations don't degrade quality metrics

### Speed Agent Queries

```typescript
// Find slowest operations (speed agent analysis)
const slow = await turso.execute(`
  SELECT agent_type, AVG(duration_ms) as avg_time
  FROM agent_executions
  WHERE status = 'success'
  GROUP BY agent_type
  HAVING avg_time > 3000
  ORDER BY avg_time DESC
`);

// Track optimization effectiveness
const optimizations = await turso.execute(`
  SELECT optimization_type,
         AVG((before_ms - after_ms) / before_ms * 100) as avg_improvement
  FROM optimizations
  GROUP BY optimization_type
  ORDER BY avg_improvement DESC
`);
```
